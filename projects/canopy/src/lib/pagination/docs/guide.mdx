## Usage

Import the component in your application:

```typescript
import { LgPaginationComponent } from '@legal-and-general/canopy';
```

In your HTML:

```html
<lg-pagination
  [totalItems]="allItems.length"
  [itemsPerPage]="itemsPerPage"
  [currentPage]="currentPage"
  (pageChanged)="onPageChanged($event)"
/>
```

And in your code:

```typescript
import { PageData } from '@legal-and-general/canopy';

export class PagingExampleComponent implements OnInit {
  allItems: unknown[] = [];
  pagedItems: unknown[] = [];
  itemsPerPage = 10;

  ngOnInit(): void {
    this.allItems = this.someService.getData();

    // Create the first page of data on init
    this.pagedItems = this.getPagedData(0, this.itemsPerPage - 1);
  }

  onPageChanged(pageData: PageData): void {
    // Respond to pageChanged events to update the displayed data
    this.pagedItems = this.getPagedData(pageData.startIndex, pageData.endIndex);
  }

  private getPagedData(startIndex: number, endIndex: number): unknown[] {
    // Note: Array.prototype.slice returns items from startIndex up to but
    // not including endIndex, so pass endIndex + 1.
    return this.allItems.slice(startIndex, endIndex + 1);
  }
}
```

## Variants

### Desktop

On large viewports (`lg` breakpoint and above) the component shows previous and next controls, always-visible first and last page buttons, the current page, up to two neighbouring pages, and ellipses to bridge any gaps. The maximum number of controls (excluding prev/next) is 7.

### Tablet

On medium viewports (`md` breakpoint) the component shows a sliding window of 5 consecutive page buttons with no ellipses, together with a results label displayed inline.

### Mobile

On small viewports (below `md`) the component shows a sliding window of 3 consecutive page buttons with no ellipses. The results label is displayed below the controls.

## Additional development details

### LgPaginationComponent inputs

| Name           | Description                                                                       | Type     | Default        | Required |
| -------------- | --------------------------------------------------------------------------------- | -------- | -------------- | -------- |
| `id`           | HTML `id` attribute, auto generated if not provided.                              | `string` | auto-generated | No       |
| `totalItems`   | Total number of items in the paged collection.                                    | `number` | `0`            | No       |
| `itemsPerPage` | Maximum number of items shown per page. Values lower than `1` are clamped to `1`. | `number` | `10`           | No       |
| `currentPage`  | Current page number. Values are clamped to the valid page range.                  | `number` | `1`            | No       |

### LgPaginationComponent outputs

| Output        | Description                                                                                                                                | Value                    |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------ |
| `pageChanged` | Emitted when the current page changes, including the requested page number and the zero-based start and end indexes for the visible slice. | `EventEmitter<PageData>` |

### PageData

| Property     | Description                                            |
| ------------ | ------------------------------------------------------ |
| `pageNumber` | The requested page number.                             |
| `startIndex` | Zero-based inclusive start index for the current page. |
| `endIndex`   | Zero-based inclusive end index for the current page.   |

### Additional notes

The component will automatically hide itself when the total number of items is less than or equal to the `itemsPerPage` input (i.e. there is only 1 page of data).

The label (Showing x-y of n results) is positioned automatically, either to the right or below the pagination buttons, depending on the available space.

The responsive layout switches automatically at the `md` and `lg` breakpoints — no additional configuration is required.
